# Bronze Layer (Raw Ingestion)

The Bronze layer is the first entry point of data into the Data Lake. Its primary goal is to retain an exact, unmodified copy of the source data while appending essential audit metadata.

## Process Flow

1. **Download (`00_setup_download.ipynb`)**:
   - Downloads the raw `.parquet` files directly from the TLC AWS CloudFront CDN.
   - Files are stored temporarily in the local `data/raw/<vehicle_type>/` directory to ensure network reliability.
   - Implements automated retries using the `tenacity` library to handle network timeouts.

2. **Ingestion (`01_bronze_ingestion.ipynb`)**:
   - PySpark reads the local `.parquet` files into memory.
   - The original schema provided by TLC is **strictly maintained**. Columns are neither dropped nor renamed at this stage.
   - An injection of a `_meta` struct is applied to every row.

## Metadata Injection (`_meta`)

To ensure data lineage and traceability back to the exact pipeline execution, the following fields are attached to the raw records before persisting to MongoDB:
- `vehicle_type`: (e.g., "yellow", "fhv").
- `run_id`: The UUID generated by the `ControlManager` for this ingestion run.
- `ingestion_time`: UTC timestamp of when the row was processed.
- `source_file`: The original file name (e.g., `yellow_tripdata_2025-01.parquet`).

## Persistence

The enriched raw data is appended to collections in the `tlc_bronze` database in MongoDB:
- `tlc_bronze.yellow_raw`
- `tlc_bronze.green_raw`
- `tlc_bronze.fhv_raw`
- `tlc_bronze.hvfhv_raw`
